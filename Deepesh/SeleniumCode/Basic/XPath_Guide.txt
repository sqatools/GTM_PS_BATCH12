1. Absolute XPath : When we follow entire path in DOM  to locate the element, then it is called Absolute XPATH.
                    e.g   /html/body/div[1]/div[1]/div[1]/div/div/div/div[2]/div/div[1]/form/div[1]/div[1]/input

2. Relative XPath : Relative XPath locate the element with the help any unique attribute value of element.
                    e.g.    //tagname[@attribute='value']              or   //*[@attribute='value']
                    ->      //input[@data-testid='royal-email']        or   //*[@data-testid='royal-email']
                    ->      //input[@name='pass']                      or   //*[@name='pass']
                    ->      //input[@placeholder="Email address or phone number"]    or   //*[@placeholder="Email address or phone number"]



Basic XPath Methods:
i)    text() method: Get element with the help of text visible on UI.
                    e.g.  //tagname[text()='value']
                      #  https://www.nseindia.com/
                     -> //a[text()='Market Data']
                     -> //a[text()='Market Turnover']

ii)   contains method: contains method help us to identify the element with help of partial values
                       ->  partial value could be a text value or attribute value.
                       e.g.  //tagname[contains(@attribute, 'partial value')]
                       ->    //input[contains(@data-testid,"royal-e")]
                       ->    //a[contains(@data-testid,"open-registration")]

                       e.g. //tagname[contains(text(), 'partial value')]
                       ->   //a[contains(text(), 'Create new account')]
                       ->   //a[contains(text(), 'new account')]
                       ->   //div[contains(text(), 'Market Statistics')]

iii)  and/or condition:  with the help of and/or condition we can locate element with multiple attribute
                         e.g. //tagname[@attrib1='value' and @attrib2='value']
                         ->   //input[@name='firstname' and @aria-label='First name']

                         e.g.  //tagname[@attrib1='value' or @attrib2='value']
                         ->    //input[@name='firstname' or @data-type='text']

iv)   starts-with: starts-with method help us to locate the element with starting value of attribute
                         e.g.   //tagname[starts-with(@attribute='value')]
                         ->     //input[starts-with(@aria-label, "Mobile number")]


# IF multiple elements having same properties then we can use indexing to locate the element
                         -> (//button[starts-with(text(), 'Sign Up')])[1]

##################### Advance XPath Methods ####################

1. parent :
          e.g.   //ref_tagname[@attrib='value']//parent::tar_tagname[@attrib='value']
          ->     //td[text()='Pune']//parent::tr
          ->     //label[text()='FROM']//parent::div/input

2. ancestor:
          e.g.   //ref_tagname[@attrib='value']//ancestor::tar_tagname[@attrib='value']
          ->     //input[@value='checkbox']//ancestor::table
          ->     //td[text()='6003']//ancestor::html//a[text()='Manual Testing']

3. following : Any web element present below side of with respect to reference element on web page
           e.g.   //ref_tagname[@attrib='value']//followings::tar_tagname[@attrib='value']
           ->     //input[@name="fromcity"]//following::input[@id="street_address2"]

4. following-sibling : Elder sibling
          e.g.   //ref_tagname[@attrib='value']//followings-sibling::tar_tagname[@attrib='value']
          ->     //table[@id='cities']//tr[5]//following-sibling::tr
          ->     //label[text()='FROM']//following-sibling::input

5. preceding : Any web element present upper side with respect to reference element on web page
           e.g.  //ref_tagname[@attrib='value']//preceding::tar_tagname[@attrib='value']
           ->    //input[@name="fromcity"]//preceding::h1

6. preceding-sibling : Younger sibling
        e.g.   //ref_tagname[@attrib='value']//preceding-sibling::tar_tagname[@attrib='value']
         ->    //table[@id='cities']//tr[5]//preceding-sibling::tr
         ->    //span[contains(text(), 'Cab booking')]//preceding-sibling::input
